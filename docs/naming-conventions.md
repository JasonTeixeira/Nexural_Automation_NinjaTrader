# Naming Conventions

## Module naming
Use **PascalCase** with descriptive names:
- AbsorptionReversal
- TrapFailureReversal
- StackedImbalanceContinuation
- DeltaDivergenceReversal
- VWAPReversion
- OpeningRangeOrderFlowBreakout

Avoid:
- marketing language
- vague names like `MyStrategyV2`

## File naming
- `README.md`, `parameters.md`, `notes.md`, `changelog.md`, `metadata.yaml`
- Keep code filenames aligned with module name.
