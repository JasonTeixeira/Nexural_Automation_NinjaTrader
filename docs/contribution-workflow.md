# Contribution Workflow

## Recommended steps
1. Fork the repository.
2. Create a branch (feature/fix/docs).
3. Copy a template into the right platform folder.
4. Fill out docs + metadata first.
5. Add code and keep changes small.
6. Add test notes.
7. Open a PR and complete the PR checklist.

## Review expectations
Maintainers will focus on:
- documentation completeness
- safety posture (no hype)
- clarity + reproducibility
- clean folder structure
